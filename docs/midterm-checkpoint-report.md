# 프로젝트 중간점검 보고서
- 작성일: 2026-03-26
- 프로젝트명: AI 기반 스터디 모임 지원 모바일 웹 서비스 (가칭: StudyFlow)
- 저장소: https://github.com/Jisu47/Prototype
- 배포 주소: https://prototype-jisu47s-projects.vercel.app

## 서론

본 프로젝트는 같은 학습 목표와 시험 일정, 발표 일정, 마감 일정을 공유하는 스터디 모임을 보다 효율적으로 운영할 수 있도록 지원하는 모바일 웹 서비스를 목표로 한다. 일반적인 스터디 모임은 공용 자료를 중심으로 함께 학습하지만, 자료가 많아질수록 어떤 내용을 언제까지 공부해야 하는지 체계적으로 정리하기 어렵고, 팀원별 학습 진도를 지속적으로 확인하는 데에도 많은 부담이 따른다. 특히 스터디 리더는 일정 정리, 자료 공유, 계획 수립, 진도 확인을 모두 수동으로 관리해야 하는 경우가 많아 운영 부담이 커지는 문제가 있다.

이 프로젝트는 이러한 문제를 해결하기 위해 공용 학습 자료와 시험 일정을 기반으로 주간 학습 계획 자동 생성, 문서 기반 AI 질의응답, 팀원별 완료 체크 및 진도 관리 기능을 하나의 흐름으로 제공하는 서비스를 지향한다. 현재 단계에서는 실제 서비스 완성본이 아닌 mock 데이터 기반의 프로토타입을 구현하여, 서비스 구조와 사용자 흐름, 화면 구성이 자연스럽게 보이는지 우선 검증하고 있다.

## 본론

### 1. 프로젝트 개요

본 서비스는 같은 시험 일정과 학습 목표를 가진 대학생 스터디 모임, 공용 PDF 및 강의자료를 함께 보는 발표 준비 모임, 같은 자료를 중심으로 학습하는 소규모 팀을 주요 사용자로 상정한다. 사용자는 모임을 생성하고 시험일 또는 마감일을 등록한 뒤, 공용 자료를 업로드하고 이를 기반으로 학습 계획을 세우며, AI 질의응답과 팀원 진도 관리를 통해 함께 공부할 수 있다.

최종 결과물의 핵심은 단순한 일정 관리 앱이 아니라, **모임이 함께 공유하는 자료를 기반으로 실제 학습을 지원하는 서비스**라는 점이다. 즉, 일정과 체크 기능만 제공하는 수준을 넘어서, 공유 자료를 바탕으로 AI가 질문에 응답하고, 시험일까지 남은 기간과 학습 목표를 반영해 공동 학습 계획을 제안하며, 팀원별 진행 상황을 함께 확인할 수 있는 협업형 학습 플랫폼을 완성하는 것이 목표이다.

현재 구현된 결과물은 이러한 최종 목표를 직접 실험해 보기 위한 프로토타입이다. 프로토타입은 실제 백엔드, 실제 RAG, 실제 파일 저장 기능 없이도 핵심 사용자 흐름과 화면 구성이 타당한지 빠르게 검증하는 데 목적이 있다.

### 2. 필요성

기존 스터디 모임은 시험 일정과 자료를 서로 공유하면서 운영되지만, 실제 학습 과정에서는 여러 가지 비효율이 반복된다. 먼저 자료는 공유되어 있어도, 어떤 자료를 어느 시점에 공부해야 하는지 체계적으로 계획하기 어렵다. 시험일까지 남은 기간과 자료 분량, 학습 목표를 종합해 학습 계획을 세우려면 많은 수작업이 필요하다.

또한 공용 자료가 쌓일수록 중요한 내용을 빠르게 파악하기 어려워지고, 팀원마다 학습 진도 차이가 발생해도 이를 한눈에 확인하고 관리하기가 쉽지 않다. 기존의 많은 스터디 앱은 일정 공유나 체크 기능에 집중되어 있어, 실제로 공유 자료를 기반으로 학습을 도와주는 기능은 부족한 경우가 많다.

따라서 본 프로젝트는 공유 자료 기반 AI 질의응답과 일정 기반 학습 계획 자동 생성 기능을 결합하여, 단순한 스터디 운영 보조를 넘어 실제 학습 지원까지 가능한 서비스가 필요하다는 문제의식에서 출발한다.

### 3. 주요 기능

본 서비스가 지향하는 주요 기능은 다음과 같다.

1. 모임 생성 및 초대
   모임장이 스터디를 생성하고, 링크 공유를 통해 팀원을 초대할 수 있다.

2. 시험 일정 및 목표 등록
   과목 또는 주제를 입력하고 시험일, 발표일, 마감일을 등록하며, 주간 또는 월간 공동 목표를 설정할 수 있다.

3. 공용 자료 업로드 및 관리
   PDF, 강의자료, 발제문 등 스터디 공용 자료를 업로드하고 팀원과 공유하며 목록을 관리할 수 있다.

4. 주간 학습 계획 자동 생성
   시험일까지 남은 기간, 업로드된 자료 수와 분량, 학습 목표를 바탕으로 모임 단위의 주간 학습 계획을 자동으로 생성한다.

5. 문서 기반 AI 질의응답
   업로드된 공용 자료를 기반으로 질문할 수 있으며, AI가 자료 내용을 바탕으로 답변을 제공한다. 이 기능은 단순한 일반 챗봇이 아니라, 공유 자료 기반 답변이라는 점이 핵심이다. 향후에는 핵심 내용 요약, 시험 대비 정리, 질문 생성 기능으로 확장할 수 있다.

6. 팀원별 완료 체크 및 진도 관리
   팀원 각자가 자신의 학습 완료 여부를 기록하고, 모임 전체 진행 현황과 팀원별 진행률을 확인할 수 있다.

### 4. 차별성

본 프로젝트의 가장 큰 차별점은 **공유 자료를 중심으로 스터디 운영과 학습 지원 기능을 통합했다는 점**이다.

첫째, 공유 자료 기반 AI 질의응답 기능을 통해 단순 일정 앱과 구분된다. 일반적인 스터디 앱은 일정 공유나 출석, 체크 중심으로 구성되지만, 본 서비스는 모임이 실제로 업로드한 자료를 바탕으로 AI가 답변하는 구조를 지향한다. 이는 개인 메모 기반 챗봇과도 다르며, 여러 사람이 함께 보는 공용 자료를 기준으로 학습을 지원한다는 점에서 의미가 있다.

둘째, 개인 플래너가 아니라 모임 전체의 시험 일정과 자료를 기준으로 공동 학습 계획을 자동 생성한다. 따라서 구성원은 각자 흩어진 방식으로 준비하는 것이 아니라, 같은 시험 범위와 같은 자료를 기준으로 함께 학습 흐름을 맞출 수 있다.

셋째, 팀원 진도 확인 및 관리 기능을 통해 스터디 리더의 운영 부담을 줄일 수 있다. 누가 어떤 항목을 완료했는지 한눈에 볼 수 있기 때문에 팀 단위 학습 운영에 적합하다.

넷째, 본 서비스는 스터디 외에도 독서 모임, 발표 준비 모임, 프로젝트 학습 모임 등 같은 자료를 함께 보는 협업형 학습 상황으로 확장 가능하다. 이 점에서 서비스 확장성도 확보할 수 있다.

### 5. 현재 구현 내용

현재 프로젝트는 가벼운 프로토타입이 완성된 상태이며, 핵심 사용자 흐름을 검증하기 위한 mock 기반 데모로 구현되어 있다. 프로토타입의 주요 흐름은 다음과 같다.

1. 모임 생성
2. 시험 일정 등록
3. 공용 자료 업로드 및 목록 확인
4. 자동 생성된 주간 학습 계획 확인
5. 문서 기반 AI 질의응답
6. 팀원별 완료 체크 및 진도 확인

구현 범위는 다음과 같다.

- 모바일 웹 형태의 프로토타입 UI
- 모임 생성 화면과 흐름
- 시험 일정 입력 흐름
- 공용 자료 업로드 UI 및 자료 목록 화면
- 자동 생성된 주간 학습 계획 화면
- 계획 완료 체크, 계획 수정, 직접 계획 추가 기능
- 문서 기반 AI 질의응답 화면
- 팀원 진도 확인 및 완료 체크 화면
- 페이지 간 이동과 상태 변화가 가능한 데모 흐름

기술적으로는 `Next.js`, `TypeScript`, `Tailwind CSS`를 기반으로 구현했고, 현재는 mock 데이터와 클라이언트 상태를 이용해 동작한다. 실제 로그인, 실제 데이터 저장, 실제 문서 업로드, 실제 RAG, 실제 AI API 연동, 실제 푸시 알림 등은 아직 포함하지 않았다. 즉, 현재 버전은 완성형 서비스가 아니라 사용자 흐름과 화면 구성을 빠르게 검증하기 위한 프로토타입이다.

또한 최근에는 홈 화면의 정보량을 줄이고 `홈 / 스터디 / 계획 / 자료` 구조로 메뉴를 단순화하여, 핵심 흐름과 차별점이 더 잘 드러나도록 개선했다. 특히 자료 메뉴 안에 자료 업로드 UI와 AI 챗봇을 함께 배치함으로써, 이 프로젝트의 핵심 기술이 공유 자료 기반 AI 지원이라는 점을 보다 직관적으로 보여주도록 구성했다.

### 6. 향후 계획

향후에는 프로토타입 단계에서 검증한 화면 구조와 사용자 흐름을 바탕으로 실제 서비스 구현을 진행할 예정이다. 기술 방향은 모바일 퍼스트 웹을 기반으로 하되, 이후 PWA 형태로 확장할 수 있도록 고려하고 있다. 배포는 `Vercel`, 백엔드 및 데이터 저장은 `Supabase`, AI 기능은 `OpenAI API` 기반으로 고도화하는 방향을 계획하고 있다.

세부적으로는 다음과 같은 확장이 필요하다.

1. `Supabase` 연동을 통한 실제 데이터 저장 구조 구축
2. 실제 문서 업로드 및 자료 관리 기능 구현
3. 일정 입력 데이터를 반영한 실제 계획 생성 로직 구현
4. `OpenAI API`를 활용한 문서 기반 AI 질의응답 기능 구현
5. 팀원별 데이터 저장과 실제 진도 관리 기능 구현
6. 추천 질문 생성, 자료 핵심 요약, 활동 통계 등 확장 기능 검토
7. PWA 기반 홈 화면 추가 및 웹 푸시 알림 기능 검토

또한 이번 프로토타입은 향후 본격적인 프로젝트 수행을 위한 기준점 역할도 한다. 즉, 프로토타입을 통해 서비스 구조와 디자인 방향, 핵심 화면 구성을 먼저 정리한 뒤, 이를 바탕으로 프론트엔드, 백엔드, AI 기능, 데이터 관리, 발표 및 문서화 등의 역할을 분담하여 본격적인 개발을 진행할 계획이다.

## 결론

본 프로젝트는 공용 자료와 시험 일정을 함께 사용하는 스터디 모임의 비효율을 해결하기 위해 기획된 AI 기반 스터디 모임 지원 모바일 웹 서비스이다. 기존 스터디 앱이 일정 공유나 단순 체크 기능에 머무르는 경우가 많은 반면, 본 서비스는 공유 자료를 바탕으로 학습 계획 자동 생성과 문서 기반 AI 질의응답, 팀원 진도 관리를 함께 제공한다는 점에서 차별성을 가진다.

현재까지는 이러한 핵심 가치를 검증하기 위한 프로토타입이 구현된 상태이며, 모바일 웹 UI와 핵심 사용자 흐름, 계획 수정 기능, 자료 기반 AI 챗봇 데모 화면 등을 통해 서비스의 방향성을 확인할 수 있다. 특히 공유 자료 기반 AI 질의응답 기능은 이 프로젝트의 핵심 기술이자 향후 완성도를 결정할 중심 요소로 볼 수 있다.

향후에는 프로토타입에서 검증한 구조를 바탕으로 실제 데이터 저장, 문서 업로드, AI 연동, 진도 관리 기능을 구현하여 서비스 수준을 높일 계획이다. 동시에 이번 프로토타입을 기준으로 화면 구성과 디자인 방향을 확정하고, 팀 내 역할 분담을 구체화하여 본격적인 캡스톤 프로젝트 개발 단계로 확장해 나갈 것이다.
